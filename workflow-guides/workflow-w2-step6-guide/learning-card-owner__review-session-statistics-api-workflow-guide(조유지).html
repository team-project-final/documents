체크리스트 항목

1. 먼저 이해할 것

이 작업은 learning-card-svc의 데이터를 생성, 조회, 변경하는 사용자 흐름과 API 계약을 구체화하는 작업이다. 테이블이나 엔드포인트를 먼저 만들기보다 누가 어떤 조건에서 어떤 데이터를 볼 수 있고 바꿀 수 있는지, 중복/권한/검증 실패를 어떤 응답으로 다룰지 먼저 이해해야 한다.

따라서 W2 Step 6을 시작할 때는 제목의 기능명만 보고 바로 코드를 작성하지 말고, 원본 WORKFLOW의 요구사항 분석, Security 검토, DTO/Entity/API 설계, 테스트 항목을 먼저 연결해서 읽는다. 그 이해를 바탕으로 TASK 문서의 Instructions, Constraints, Output Format을 갱신한 다음 구현과 검증을 진행한다.

이 문서의 목적

이 가이드는 처음 프로젝트를 접한 개발자가 WORKFLOW_learning-card_W2.md의 W2 Step 바로 진행할 수 있도록 원본 workflow 체크리스트를 실행 순서로 재구성한 문서다.

Done When 기준

아래 체크리스트가 완료되고, 관련 테스트·빌드·실행 확인 결과가 PR 산출물에 포함되면 Step 완료로 판단한다.

진행 체크리스트

6.1 TASK 시작

  • Step Goal / Done When / Scope / Input 확인
  • PRD_W2 해당 요구사항 확인 (복습 통계)
  • Duration 산정 확인

6.2 요구사항 분석

  • 일별 복습 수 집계 요건 (최근 N일)
  • 주별 복습 수 집계 요건 (최근 N주)
  • 정답률 계산 기준 정의 (Good+Easy / 전체)
  • 통계 데이터 응답 포맷 정의
  • Instructions 초안 → TASK 문서 반영

6.3 Security 1차 검토

  • 인증 필요 여부: Yes (JWT 인증 필요)
  • 권한 종류: 로그인 사용자 (본인 통계만)
  • 공개 API 여부: No
  • 결과 → TASK Constraints 반영

6.4 ERD 설계

  • 추가 테이블 불필요 (review_logs + review_sessions 기반 집계)
  • 집계 쿼리 설계 (GROUP BY date/week, COUNT, 정답률 계산)
  • 인덱스 최적화 확인 (review_logs.userId + reviewedAt)
  • Duration(final) 갱신

6.5 Security 2차 검토

  • 통계 데이터 접근 제어 (본인만)
  • 다른 사용자 통계 접근 불가 확인
  • 결과 → TASK Constraints 반영

6.6 DTO / Entity 설계 (API First)

  • DailyReviewStatResponse 정의 (date, reviewCount, correctRate)
  • WeeklyReviewStatResponse 정의 (weekStart, reviewCount, correctRate)
  • ReviewStatsResponse 정의 (daily[], weekly[], totalReviews, overallCorrectRate)
  • Output Format → TASK 반영

6.7 Repository 구현

  • ReviewLogRepository 커스텀 쿼리 추가
  • 일별 집계 쿼리 (GROUP BY DATE(reviewedAt), COUNT, 정답률)
  • 주별 집계 쿼리 (GROUP BY WEEK, COUNT, 정답률)
  • Native Query 또는 JPQL 작성

6.8 Service + Test

  • ReviewStatsService 구현 (getDailyStats, getWeeklyStats, getOverallStats)
  • 일별 복습 수 + 정답률 계산 로직
  • 주별 복습 수 + 정답률 계산 로직
  • 기간 필터링 (최근 30일 / 12주)
  • 데이터 없는 날짜 0으로 채우기 로직
  • 단위 테스트 작성 (Mockito)
  • 경계값 테스트 (데이터 없는 기간, 정답률 0%)
  • 테스트 통과 확인

6.9 Controller + Test

  • GET /api/v1/review/stats/daily?days={n} 엔드포인트 구현
  • GET /api/v1/review/stats/weekly?weeks={n} 엔드포인트 구현
  • GET /api/v1/review/stats 엔드포인트 구현 (종합)
  • 슬라이스 테스트 (@WebMvcTest)
  • 401/403 응답 테스트
  • 통합 테스트 (복습 데이터 → 통계 API 검증)
  • 테스트 통과 확인

6.10 View + Test (해당 시)

  • Flutter 화면 연동: 해당 없음 (프론트 별도)
  • Swagger API 문서 확인
  • RULE Reference → TASK 반영

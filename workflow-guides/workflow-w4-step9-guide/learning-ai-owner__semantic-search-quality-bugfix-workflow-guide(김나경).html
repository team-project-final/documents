진행 기준

W4는 기능 추가보다 E2E 검증, 버그 수정, 성능/배포 안정화가 중심이다. TASK의 Done When과 PRD_W4 완료 기준을 먼저 확인하고, 실패 케이스는 재현 절차와 수정 결과를 함께 남긴다.

위험/릴리즈 체크

  • 릴리즈 전 인증/권한, 성능, 장애 대응, 롤백 기준을 확인한다.

검증 우선순위

  • 시맨틱 검색 정확도 검증 시나리오 목록 작성
  • 테스트 데이터 준비
  • 시맨틱 검색 정확도 검증 테스트 실행
  • 실패 항목 기록
  • P0/P1/P2 분류
  • P0 즉시 수정 대상 확정
  • P0 버그 수정
  • 시맨틱 검색 관련 버그 수정
  • 수정 코드 리뷰 + 테스트
  • 수정 후 전체 테스트 재실행

  • 원본 WORKFLOW의 모든 체크 항목이 테스트 결과, 수정 커밋, 운영 문서 중 하나로 추적되는지 확인한다.

작업 이해

이 작업은 체크리스트를 단순히 순서대로 처리하는 것이 아니라, 먼저 해당 Step이 어떤 사용자 가치와 시스템 책임을 검증하거나 구현하려는지 이해한 뒤 진행해야 한다. 시작 전에 PRD, TASK, WORKFLOW의 목표와 완료 기준을 연결해서 읽고, 이번 Step의 산출물이 이전 주차 결과물과 다음 Step에 어떤 영향을 주는지 확인한다.

처음 프로젝트를 접한 개발자는 기능명만 보고 바로 구현하지 말고, 담당 서비스의 경계, 입력/출력 계약, 보안 제약, 테스트 기준을 먼저 정리한다. 이해한 내용을 TASK의 Instructions 또는 Constraints에 반영한 다음 설계, 구현, 검증을 진행하는 것을 기본 흐름으로 삼는다.

이 문서의 목적

이 문서는 frontend-owner가 Flutter Web 프로젝트의 기본 구조를 생성하고, ProviderScope, GoRouter, ThemeData가 적용된 빈 화면 라우팅까지 확인하는 절차를 설명한다.

Done When

flutter run -d chrome으로 앱이 실행되고, /, /login, /dashboard 라우팅 전환이 동작하며, DESIGN.md 기반 ThemeData가 적용되면 완료로 본다.

착수 전 확인

TASK는 Flutter 3.24+ / Dart 3.5+, Riverpod 2.x를 기준으로 한다. wiki 18번 기술 문서의 버전 매핑에는 Flutter 3.41.x / Dart 3.8+ 및 Riverpod 3.x 언급도 있으므로 실제 pubspec.yaml 작성 전 팀 기준 버전을 확정한다.

WORKFLOW 진행 순서

1TASK 시작

  • Step Goal, Done When, Scope, Input을 확인한다.
  • PRD_W1 프로젝트 골격 요구사항을 확인한다.
  • Duration 1일을 확인한다.

2요구사항 분석

  • Flutter 3.24+ / Dart 3.5+ 구조를 확인한다.
  • Riverpod + GoRouter + Material 3 + google_fonts 의존성을 확정한다.
  • DESIGN.md의 ColorScheme, Typography 기준을 반영한다.

3Security 1차 검토

  • 골격 단계이므로 인증은 구현하지 않는다.
  • 권한 종류는 없다.
  • 프론트엔드 앱이므로 공개 API는 만들지 않는다.

4ERD 설계

  • 프론트엔드 단계라 ERD는 비해당이다.
  • lib/core, lib/features, lib/shared 구조를 만든다.
  • /, /login, /dashboard 라우트를 정의한다.

5Security 2차 검토

  • 민감 정보 암호화는 비해당으로 둔다.
  • .env와 API base URL 관리 위치를 정한다.
  • 코드 내 시크릿 하드코딩 금지를 확인한다.

6DTO / Entity 설계

  • DTO와 Entity는 만들지 않는다.
  • 빈 Scaffold 페이지만 생성한다.
  • GoRouter와 ThemeData를 앱에 연결한다.

7Repository 구현

  • pubspec.yaml에 의존성을 추가한다.
  • ProviderScope로 앱 최상위를 감싼다.

8Service + Test

  • GoRouter 설정을 구현한다.
  • ThemeData의 ColorScheme + Typography를 구현한다.
  • 라우팅 전환 Widget 테스트를 작성한다.

9Controller + Test

  • flutter run -d chrome 실행을 확인한다.
  • 라우팅 전환과 DESIGN.md 테마 적용을 확인한다.

10View + Test

  • 각 라우트 빈 화면 렌더링을 확인한다.
  • Smoke Test 1건을 남긴다.
  • RULE Reference를 TASK에 반영한다.

구현 예시

프로젝트 생성과 의존성

flutter create --platforms web synapse_frontend
cd synapse_frontend

flutter pub add flutter_riverpod go_router google_fonts
flutter pub get

pubspec.yaml 핵심 의존성

dependencies:
  flutter:
    sdk: flutter
  flutter_riverpod: ^2.6.1
  go_router: ^14.0.0
  google_fonts: ^6.1.0

main.dart

import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'app/app.dart';

void main() {
  runApp(const ProviderScope(child: SynapseApp()));
}

app.dart

import 'package:flutter/material.dart';
import '../core/theme/app_theme.dart';
import 'router.dart';

class SynapseApp extends StatelessWidget {
  const SynapseApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp.router(
      title: 'Synapse',
      theme: AppTheme.light(),
      routerConfig: appRouter,
      debugShowCheckedModeBanner: false,
    );
  }
}

router.dart

import 'package:go_router/go_router.dart';
import '../features/home/home_page.dart';
import '../features/auth/login_page.dart';
import '../features/dashboard/dashboard_page.dart';

final appRouter = GoRouter(
  routes: [
    GoRoute(path: '/', builder: (_, __) => const HomePage()),
    GoRoute(path: '/login', builder: (_, __) => const LoginPage()),
    GoRoute(path: '/dashboard', builder: (_, __) => const DashboardPage()),
  ],
);

ThemeData

import 'package:flutter/material.dart';
import 'package:google_fonts/google_fonts.dart';

class AppTheme {
  static ThemeData light() {
    const seed = Color(0xFFD97706);

    return ThemeData(
      useMaterial3: true,
      colorScheme: ColorScheme.fromSeed(
        seedColor: seed,
        brightness: Brightness.light,
      ),
      textTheme: GoogleFonts.plusJakartaSansTextTheme(),
      scaffoldBackgroundColor: const Color(0xFFFAFAF9),
    );
  }
}

빈 Scaffold 페이지

import 'package:flutter/material.dart';

class DashboardPage extends StatelessWidget {
  const DashboardPage({super.key});

  @override
  Widget build(BuildContext context) {

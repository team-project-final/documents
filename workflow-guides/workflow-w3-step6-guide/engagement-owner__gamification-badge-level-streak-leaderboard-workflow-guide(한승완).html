1. 먼저 이해할 것

이 작업은 점수나 배지를 보여주는 UI 이전에 사용자 활동을 보상 규칙으로 해석하는 engagement-svc 작업이다. XP 적립, 레벨업, 스트릭, 배지 수여, 리더보드 노출은 조작 방지와 개인정보 노출 최소화가 중요하므로 서버 기준 규칙과 공개 범위를 먼저 이해해야 한다.

따라서 W3 Step 6을 시작할 때는 제목의 기능명만 보고 바로 코드를 작성하지 말고, 원본 WORKFLOW의 요구사항 분석, Security 검토, DTO/Entity/API 설계, 테스트 항목을 먼저 연결해서 읽는다. 그 이해를 바탕으로 TASK 문서의 Instructions, Constraints, Output Format을 갱신한 다음 구현과 검증을 진행한다.

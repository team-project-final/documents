| **What** | - **Step 1: 레포 부트스트랩** (`README.md`, `.github/CODEOWNERS`, `.gitignore`, `.editorconfig` 초기 커밋, 2026-05-12) <br> - **Step 2: parse-workflow CI 개선** — push 트리거를 `main` 한정에서 전체 브랜치로 확장하고 대시보드 push 단계에 `pull --rebase` + 3회 재시도 로직 추가 (2026-05-14) <br> - **Step 3: Gradle wrapper 실행권한 수정** (`gradlew` 파일 모드 변경, 2026-05-15) <br> - **Step 4: dev 프로파일 DDL 보정** — `application-dev.yml`에 `ddl-auto: update` 추가 (2026-05-21, PR #26) <br> - **Step (인프라): application.yml 표준화 + dev-smoke CI (port 8081)** — 앱명 `synapse-platform-svc`→`platform-svc`, 기본 프로파일 local→dev, JWT prefix `jwt.*`→`synapse.jwt.*`, local yml 제거, prod Hikari pool 20+redis(env), CI dev 추가 + `docker-compose.ci.yml`(postgres+redis) dev-smoke(8081) (PR #34) |
| **Why** | 부트스트랩 커밋으로 모듈 구성·기술 스택(Java 21, Spring Boot, Spring Modulith)을 README에 명시해 프로젝트 골격을 세웠고, parse-workflow는 docs 변경을 `_dashboard`로 모으는 자동화라 모든 브랜치 작업을 대시보드에 반영하려 트리거를 확장했다. dev 보정 커밋 본문은 "Fresh RDS에서 Flyway migration 후 JPA validate 실패 방지"를 명시한다. |
| **Problem** | (1) 대시보드 push가 동시 작업 시 충돌해 실패할 수 있었음(Step 2의 retry 도입이 근거). (2) 커밋 본문 D-024에 기록된 실제 장애 — Fresh RDS 환경에서 `mfa_credentials` 테이블이 없어 JPA validate가 실패함(Step 4). 추가로 dev `synapse.jwt.private-key` 기본값이 손상된 base64라 public-key와 쌍 불일치 → 토큰 라운드트립 실패. |
| **Solution** | (1) push 단계를 `git pull --rebase origin main && git push`를 최대 3회 시도하는 루프로 바꿔 동시성 충돌을 흡수. (2) dev 환경에 한해 `ddl-auto: update`를 적용해 Flyway 이후 누락 테이블을 JPA가 자동 생성하도록 보정(운영이 아닌 dev 프로파일로 범위 한정). 추가로 private-key 기본값을 public-key와 짝 맞는 유효 PKCS8 키로 교정. |
| **회고** | `fix(ci)`·`fix(dev)` 커밋이 연이어 나온 것으로 보아 CI 파이프라인과 Fresh DB 부팅 같은 환경 이슈를 실사용 중 발견해 즉시 좁은 범위로 패치하는 흐름이 잘 작동했다. 특히 dev 한정 `ddl-auto`와 push 재시도처럼 운영 안전성을 해치지 않는 선에서 문제를 격리해 해결한 점이 돋보인다. 커밋 본문에 D-024 같은 이슈 ID와 원인을 남겨 추적성을 확보한 습관도 강점이다. 추가로 JWT 키쌍은 prefix 표준화와 키 유효성·키쌍 일치를 함께 검증해야 하며, dev-smoke 부팅 검증이 손상 키 기본값 같은 설정 결함을 조기에 드러낸다. <br><br> **06-02~06-08(직접 커밋 없음 — 감사·위임)**: `synapse.kafka.enabled` 게이트는 #59로 **해소(G1)**, Flyway 가드 caller는 main+dev 모두 적용(**4서비스 중 가장 표준 정합**, G5). platform-svc의 **V28 중복**(머지된 `allow_multiple_refresh_tokens` vs untracked `rename_oauth_provider_id_column`)이 06-05 Flyway 타임스탬프 표준 수립의 발단이었고, 후속 정리는 이슈 #65로 위임. 잔여: Avro/serializer 1.12.0/7.7.0이 정본 초과(G4). |

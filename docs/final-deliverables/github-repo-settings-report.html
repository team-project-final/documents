핵심 확인 결과

  • 모든 레포의 기본 브랜치는 main이고, archived 상태인 레포는 없습니다.
  • 모든 레포에서 GitHub Actions가 활성화되어 있으며 허용 정책은 all입니다.
  • 모든 레포의 기본 브랜치 보호는 활성화되어 있지만, 필수 status check가 설정된 레포는 없습니다.
  • PR 리뷰 필수 설정은 9개 레포에 적용되어 있습니다. 서비스 레포들은 대체로 1명 승인, stale review dismiss, CODEOWNERS 리뷰 요구가 켜져 있습니다.
  • auto-merge는 모든 레포에서 꺼져 있습니다. 브랜치 삭제 자동화는 8개 레포에서만 켜져 있습니다.
  • private 레포는 demo-repository, synapse-gitops, synapse-mirror 3개입니다.

권장 점검 항목

  • 필수 CI 상태 체크: 모든 레포에서 비어 있습니다. PR 병합 전 테스트나 빌드가 필요하다면 required status checks를 추가하는 것이 맞습니다.
  • Actions 허용 범위: 모든 레포가 all입니다. 외부 Action 사용을 제한해야 하는 프로젝트라면 org/repo 정책을 조정하세요.
  • 병합 전략 통일: 서비스 레포는 squash-only에 가깝지만 문서/프로토타입 레포는 merge, squash, rebase가 모두 열려 있습니다. 운영 정책을 정하면 이 설정을 맞추는 편이 좋습니다.
  • 브랜치 자동 삭제: 일부 레포만 delete_branch_on_merge가 켜져 있습니다. PR 브랜치 정리를 자동화하려면 나머지 레포도 활성화하세요.

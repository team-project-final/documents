## 1. 추가 · 보충 · 보완 기능 정리

### 1.1 [추가] 데모 전용 — 스펙·프론트·API 모두 근거 없음 → **채택 여부 의사결정 필요**

| 기능 | 화면 | 가치 | 채택 시 필요 작업 |
|------|------|------|-------------------|

| **QA 좋아요/싫어요 피드백** | QA | 답변 품질 개선 루프 | 피드백/평점 API 신설 |

| **mega-dashboard 집약** | 홈 | 한눈 파악 | IA 결정 사항(§6.5). 게이미피케이션 절제 원칙과 충돌(§6.3) |

### 1.2 [보완] 스펙·프론트엔 있으나 데모에 빠진 것 — 데모 보강 후보

- **인증:** MFA 인증, OAuth 동의 화면
- **노트:** 폴더 트리/중첩, 태그 관리, 미생성 링크→노트 생성, AI 요약, 로컬 그래프 임베드
- **카드:** 카드 목록 브라우저, 카드 생성/편집(Basic/Cloze), AI 카드 생성, 카드 이동/복사(다중선택), **세션 결과 화면**
- **그래프:** 노드 선택 액션 패널(열기/AI생성/확장), 클러스터 뷰, 이웃 확장
- **커뮤니티:** 공유덱 상세·평점(1~5)·복사, 공유노트 탐색, 신고, 멤버관리, 그룹 생성/편집
- **설정:** 보안/MFA, 알림(카테고리별 push/email/in_app + 방해금지), 데이터/GDPR(export/import/삭제), 테넌트, **설정 하위 네비게이션**
- **공통:** CommandPalette(Cmd+K)
- **큰 도메인:** 결제, 관리자 콘솔, 알림 센터, 게이미피케이션 전용(배지/리더보드/레벨업 모달)

### 1.3 [보충] 명칭·범위 정합 정리(혼선 방지)

- 데모 "AI **튜터**" ≡ 스펙 "AI **Q&A** (SEARCH-002)". 명칭 통일 필요.
- 데모 노트 상세 "AI에게 질문" ↔ 스펙 노트측 CTA "AI 카드 생성". 두 CTA를 **병존**시킬지 결정.
- 데모 "카테고리" 색상 그룹 ↔ 스펙 "클러스터(자동 토픽 군집)". 동일 개념인지 정의 필요.
- 그래프 진행률 "링" vs 스펙 "바" 등 경미한 표현 차이는 디자인 시스템 컴포넌트 표준에 흡수.

## 2. 데모 전용 기능의 데이터 정합성 메모 (이전 시각 검토에서 발견)

- 온보딩 체크리스트가 **0/3 미완료**인데 계정은 Lv7·노트 24·스트릭 14일 → 데모 시드 데이터 모순.
- 홈 "18장 복습 대기" vs 덱 합계(12+5+20=37) 불일치.

- 프리뷰 신뢰도를 위해 시드 데이터 일관성 점검 권장(데모 한정 이슈).

## 3. 디자인 통일성 검토

> **결정 방식:** 본 문서는 어느 쪽이 옳다고 단정하지 않고, **amber 시스템 ↔ 데모(보라) 방향을 병기**하여 팀 의사결정에 부친다(§7).

### 3.1 [최우선] 컬러 아이덴티티 — amber(합의/구현) vs 보라(데모)

| 항목 | `DESIGN.md` + 프론트 구현 | 데모 |
|------|---------------------------|------|
| 액센트 | **Warm Amber `#D97706`** (hover `#B45309`, light `#FEF3C7`) | **보라/바이올렛** (CTA·선택·로고) |
| 보조 | Muted Teal `#0D9488` (성공/그래프) | 마젠타/핑크 등 다색 |
| 중립 | Warm **Stone** 계열(`#FAFAF9`~`#1C1917`) | 쿨/뉴트럴 그레이 + 라일락 톤 |
| 원칙 | **Restrained**(1 액센트 + 중립, 색은 드물고 의미있게) | 다색(그래프 6색, 복습 4색 버튼, 그라디언트) |
| 그라디언트 | 배경 그라디언트 **금지**(XP 바 amber 그라디언트만 허용) | 로고·AI 스파클·일부 요소 그라디언트 사용 |
| 의사결정 로그 | 2026-05-07: *"StockPilot(녹)/DevPath(파)/StudyMate(잉크블루)와 차별화, **Obsidian(보라/다크) 회피**"* 명시 | — |

- **충돌의 본질:** 데모의 보라는 `DESIGN.md`가 *명시적으로 피하려던* 룩(Obsidian)이다. 형제 프로젝트 차별화 전략과도 정면 배치.
- **프론트 영향:** `lib/core/theme/app_colors.dart`(amber/stone), `app_theme.dart`(`ColorScheme.fromSeed(primaryAmber)`)가 이미 amber로 구현됨 → 보라 채택 시 전면 리스킨.

### 3.2 타이포그래피 — 3자 모두 다름

| 소스 | Display | Body |
|------|---------|------|
| `DESIGN.md` | **Fraunces**(serif, 차별점) | Plus Jakarta Sans / Geist Mono |
| 프론트 구현 | (Fraunces 미적용) | **Noto Sans**(`google_fonts notoSansTextTheme`) |
| 데모 | 굵은 산세리프(Pretendard/Noto 계열 추정) | 동일 산세리프 |

- **공통 이탈:** 디스플레이 세리프(Fraunces) 부재. **한글 콘텐츠라 Fraunces/Plus Jakarta는 한글 글리프가 없어 비현실적** → `DESIGN.md` 타이포 섹션을 한글 폰트(Pretendard 등) 기준으로 **현실화하는 별도 결정** 필요(데모/프론트 무관하게 갱신 대상).

### 3.3 게이미피케이션 표현 — `DESIGN.md` 원칙 위반

- `DESIGN.md`: *"절제, 네온/번쩍임/그라디언트 배경 금지, **XP·레벨·순위가 동시에 3개 이상 눈에 띄면 안 됨**, 도서관의 훈장 느낌"*.
- 데모 홈: **주간 인사이트(XP +420) + 스트릭 14일 + Lv7 진행바 + 그룹 랭킹**을 한 화면에 동시 노출 → "3개 이상 금지" 원칙 위반. 색 대비도 게임형에 가까움.

### 3.4 기타 시각 품질(이전 검토에서 발견, 방향과 무관하게 유효)

- **명도 대비(WCAG AA 4.5:1)**: placeholder·카드 부제·사이드바 하단(관리자/알림/설정) 회색 텍스트가 기준 미달 가능 → 한 단계 진한 톤 권장.
- **여백 활용**: 노트 라이브러리(24개 중 3개만 + 큰 공백), 설정 프로필(데스크톱 ~60% 공백), QA 좌측 여백 → `max content width 1280px` + 정렬/필러로 밀도 조정.
- **권한 게이팅**: 학습자(Lv7) 사이드바에 "관리자" 진입점 노출 → 역할 기반 숨김(스펙은 별도 Admin Shell).
- **부합 항목(양호)**: 둥근 카드·subtle shadow·Comfortable 여백은 `DESIGN.md` 컴포넌트/spacing 방향과 부합.

### 3.5 정보구조(IA) 통일성

- 스펙·프론트 = **런처형 대시보드 + 전용 화면**(그래프/게이미피케이션/통계/AI가 각 destination).
- 데모 = **mega-dashboard**(여러 전용 화면을 홈에 흡수) + 신규 "플래너" 탭.
- → 데모 IA는 스펙뿐 아니라 **이미 구현된 프론트 라우트 구조와도 불일치**. 어느 IA를 표준으로 삼을지 결정 필요.

## 4. 의사결정 안건 (Decision Agenda)

### 안건 A — 디자인 방향

| | **A안: 기존 amber 시스템 기준** | **B안: 데모(보라) 리브랜딩 채택** |
|---|---|---|
| 내용 | 데모를 amber/stone/절제 시스템으로 재디자인 | 보라+튜터 비주얼을 정식 채택 |
| 영향 범위 | 데모 측 리디자인만 | `DESIGN.md`(컬러/모션/게이미피케이션/결정로그) + 프론트 `app_colors/app_theme` + **이미 만든 ~30개 화면 전면 리스킨** |
| 장점 | 합의·구현·형제 차별화 전략 유지, 비용 최소 | 데모의 높은 시각 완성도 유지 |
| 단점 | 데모 보라 비주얼 포기 | 차별화 전략(Obsidian 회피) 무력화, 대규모 리스킨 비용 |
| 권고 | (병기 — 팀 결정) | (병기 — 팀 결정) |

### 안건 B — 튜터 신규 기능 정식 편입 여부

플래너 / AI 단계적 힌트 / AI 추천 퀴즈 / 대화 이력 / QA 피드백 / AI FAB 을 `06`(화면)·`07`(요구사항)·`04`(API)에 **정식 반영할지** 결정. 채택 시 신규 엔드포인트 설계 필요(§4.1 표).

### 안건 C — 명칭·IA 표준

- "AI 튜터" ↔ "AI Q&A" 명칭 통일.
- mega-dashboard ↔ 런처형 대시보드 중 표준 IA 확정.
- "관리자"는 학습자 네비에서 분리(별도 Admin Shell) 확정.

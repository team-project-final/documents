## 핵심 서비스 관점

### Platform

`synapse-platform-svc`는 Synapse의 인증 허브이자 플랫폼 공통 기능 서비스다. 인증, 사용자, 결제, 알림, 감사 로그, 관리자 기능을 Spring Modulith 모듈 경계 안에서 관리한다. 전 서비스가 사용하는 JWT 검증 체계의 기준점이며 PostgreSQL, Redis, Kafka/Avro, Stripe, FCM, AWS SES 같은 외부/인프라 의존성이 집중되어 있다.

### Gateway

`synapse-gateway`는 외부 요청의 첫 진입점이다. Spring Cloud Gateway WebFlux 기반으로 JWT 검증, 공개 경로 관리, CORS, Rate Limiter, 백엔드 라우팅, 사용자/테넌트 식별 정보 전달을 담당한다.

### Knowledge

`synapse-knowledge-svc`는 노트와 지식 그래프, 청킹, 검색 관련 기능을 담당한다. Spring Boot 기반이며 JPA, Elasticsearch, Redis, Kafka, Avro, Flyway를 사용한다. README는 초기 상태지만 build 설정과 소스 구조상 실제 서비스 구현이 진행되어 있다.

### Learning

`synapse-learning-svc`는 카드/SRS 학습 기능과 AI 기능이 함께 있는 도메인이다. `learning-card`는 Java/Spring 계열 서비스이고, `learning-ai`는 FastAPI 기반 Python 서비스로 OpenAI/Anthropic SDK, SQLAlchemy, pgvector, Kafka, Redis 등을 사용한다.

### Engagement

`synapse-engagement-svc`는 커뮤니티와 게이미피케이션 도메인을 담당한다. 커뮤니티 그룹/멤버/신고 기능, XP/레벨/배지/리더보드, Kafka 이벤트 소비/발행, DB 마이그레이션과 테스트가 포함되어 있다.

## 프론트/데모 관점

`synapse-frontend`는 Flutter 기반 메인 클라이언트다. README 기준 현재는 디자인 프로토타입이며 mock 데이터로 화면/UX 흐름이 동작한다. 실제 백엔드 연동은 각 feature의 TODO 지점에 남아 있다.

`synapse-prototype`은 React/Vite 기반 인터랙티브 데모와 Living Documentation 사이트다. localStorage를 사용해 노트 작성, AI 카드 생성, 복습, XP 적립이 실제처럼 동작하는 시뮬레이션을 제공한다.

`synapse-flow-simulator`, `moking-data-guide`, `workflow-guide`, `schedule-repo`는 프로젝트 설명, 목킹 전략, 일정/흐름 시각화, 학습용 시뮬레이터 성격의 보조 웹 자산으로 볼 수 있다.

## 인프라/운영 관점

`synapse-shared`는 서비스 간 계약과 로컬 통합 환경의 중심이다. Avro 스키마, Kafka 샘플 이벤트, Docker Compose, 테스트 시드, Flyway 가드, Schema Registry 검증 스크립트, 운영 룰/리포트 문서를 포함한다.

`synapse-gitops`와 `synapse-gitops-s6`는 Kubernetes 배포 자동화 저장소다. ArgoCD ApplicationSet, Kustomize base/overlay, Terraform 기반 AWS/EKS dev 인프라, 모니터링/Ingress/External Secrets 구성, 문서 포털을 포함한다.

`workflow-dashboard`는 팀 진행 상황을 GitHub Pages에서 확인하기 위한 정적 대시보드다. 서비스별 PRD/TASK/WORKFLOW/변경 이력을 JSON과 React UI로 보여주는 용도다.

## 문서/온보딩 관점

`documents`, `documents.wiki`, `docs`, `synapse-onboarding`은 프로젝트 이해와 인수인계를 위한 문서 계층이다.

- `documents`: 의사결정, 워크플로우, 온보딩, 프로젝트 관리 문서 저장소
- `documents.wiki`: GitHub Wiki 형식의 서비스별 아키텍처 문서
- `docs`: 루트 차원의 아키텍처 상태 보고서와 작업 계획/스펙
- `synapse-onboarding`: 신입 개발자가 전체 시스템 흐름을 학습하는 Flutter Web 포털

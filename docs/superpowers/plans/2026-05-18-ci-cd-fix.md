## Task 3: synapse-learning-svc — Fix ci.yml build-java job

**Files:**
- Modify: `.github/workflows/ci.yml`

The current `build-java` job runs `./gradlew build` from repo root, but `gradlew` lives in `learning-card/`. Also need to add Modulith verify (was only in the deleted ci-java.yml).

- [ ] **Step 1: Open ci.yml and replace the build-java job**

Replace the entire `build-java` job block. The current content is:

```yaml
  build-java:
    needs: detect-changes
    if: needs.detect-changes.outputs.java == 'true'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '21'
      - uses: gradle/actions/setup-gradle@v4
      - name: Build & Test (Java)
        run: ./gradlew build
```

Replace with:

```yaml
  build-java:
    needs: detect-changes
    if: needs.detect-changes.outputs.java == 'true'
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: learning-card
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '21'
      - uses: gradle/actions/setup-gradle@v4
      - name: Grant execute permission
        run: chmod +x gradlew
      - name: Build & Test
        run: ./gradlew build --no-daemon
      - name: Modulith verify
        run: ./gradlew test --tests '*ModuleStructureTest' --no-daemon
```

Key changes:
- Added `defaults.run.working-directory: learning-card`
- Added `chmod +x gradlew` step
- Added `--no-daemon` flag
- Added Modulith verify step (migrated from deleted ci-java.yml)

- [ ] **Step 2: Verify YAML syntax**

Run: `python3 -c "import yaml; yaml.safe_load(open('.github/workflows/ci.yml'))"`
Expected: no error output

- [ ] **Step 3: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "fix(ci): add working-directory and chmod to build-java job

gradlew lives in learning-card/, not repo root.
Also migrated Modulith verify step from deleted ci-java.yml."
```

## Task 5: synapse-learning-svc — Fix ruff lint violations

**Files:**
- Modify: `learning-ai/**/*.py` (exact files determined by ruff output)

- [ ] **Step 1: Install dev dependencies**

```bash

source .venv/bin/activate
pip install -e '.[dev]'
```

- [ ] **Step 2: Run ruff to see all violations**

Run: `ruff check .`
Expected: list of violations with file paths and rule codes. Note each violation.

- [ ] **Step 3: Auto-fix safe violations**

```bash
ruff check --fix .
```

This handles import sorting (I), unused imports (F401), and other safe auto-fixes.

- [ ] **Step 4: Review the diff from auto-fix**

Run: `git diff`
Expected: import reordering, unused import removal. Verify no behavioral changes — especially check that B (bugbear) and SIM (simplify) fixes didn't alter logic.

- [ ] **Step 5: Run ruff again to check remaining violations**

Run: `ruff check .`
Expected: either clean (exit 0) or remaining violations that need manual fix.

- [ ] **Step 6: Manually fix any remaining violations**

For each remaining violation, edit the file to resolve it. Common patterns:
- `B006`: mutable default argument → use `None` default + assign in body
- `SIM108`: ternary instead of if/else → rewrite as ternary
- `UP`: use modern Python syntax (f-strings, type unions, etc.)

- [ ] **Step 7: Verify ruff passes clean**

Run: `ruff check .`
Expected: exit 0, no output

- [ ] **Step 8: Verify mypy passes**

Run: `mypy app`
Expected: `Success: no issues found` (or pre-existing issues unrelated to our changes)

- [ ] **Step 9: Commit**

